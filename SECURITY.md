# Security Policy

## Supported Versions

Only the [latest stable release](https://github.com/seatsurfing/seatsurfing/releases) is supported. Please make sure you're upgrading regularly.

## Reporting a Vulnerability

If you find a vulnerability, please **do not** report it publicly via GitHub Issues or Pull Requests. Instead, please contact [security@seatsurfing.io](mailto:security@seatsurfing.io) directly. We'll look into it shortly and fix it asap.

## Bug Bounty

We may offer a monetary reward for responsibly disclosed vulnerabilities. The decision on whether a reward is granted, its amount, as well as the payment terms and payment method are made solely at the discretion of Seatsurfing GmbH.

To be eligible for a reward, the following conditions must all be met:

- The vulnerability must be present in the [latest stable release](https://github.com/seatsurfing/seatsurfing/releases).
- The vulnerability must not have been previously disclosed or publicly known.
- The vulnerability must be practically exploitable (i.e. a real-world attack scenario must be feasible).
